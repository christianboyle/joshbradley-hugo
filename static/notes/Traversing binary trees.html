Traversing binary trees

I am still working to understand all that can be learned about the data while traversing a binary tree, but some that I have learned through study are as follows.

Breadth-first and Depth-first search

The difference between depth-first and breadth-first search is that depth-first uses a stack (last in first out) while breadth-first uses a queue (first in first out).

Breadth-first search generally takes more memory than depth-first search, but is guaranteed to find the shortest path to an element.

Iterative approach

function dfs(rootNode)  {
  const nodes = [];
  nodes.push(rootNode);

  while (nodes.length) {
    const node = nodes.pop();
    console.log(node.value);
    if (node.left) nodes.push(node.left);
    if (node.right) nodes.push(node.right);
  }
}

function bfs(rootNode) {
  const nodes = [];
  nodes.push(rootNode);

  while (nodes.length) {
    const node = nodes.shift();
    console.log(node.value);
    if (node.left) nodes.push(node.left);
    if (node.right) nodes.push(node.right);
  }
}

Keep in mind, shift() is inefficient because it is an O(n) operation compared to O(1) for pop(). Because of this it is better to implement your own queue (which can be done using two arrays/stacks).

Recursive approach

function dfs(node) {
  console.log(node.value);
  if (node.left) dfs(node.left);
  if (node.right) dfs(node.right);
}

// Because bfs is a queue, and recursive functions form a call stack
// it doesn't make much sense to perform bfs recursively

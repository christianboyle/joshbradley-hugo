Proving the correctness of algorithms

Three conditions must be defined and satisfied by the unchanging properties of the loop, kown as a loop invariant, for an algorithm to be proved correct.

Initialization: The loop invariant is true before the first iteration of the loop.

Maintenance: The loop invariant established in initialization must remain true before each iteration.

Termination: When the loop terminates, the loop invariant proves the algorithm's correctness.

A helpful example is to establish the loop invariant for insertion sort and show how initialization, maintenance, and termination prove it to be correct.

Insertion sort

for j = 2 to A.length
  key = A[j]
  i = j - 1
  while i > 0 and A[i] > key
    A[i + 1] = A[i]
    i = i - 1
  A[i + 1] = key

Loop invariant

At the start of each iteration of the outer for loop, the subarray A[1...j-1] contains the original elements of A[1...j-1] in sorted order.

Initialization

Before the first loop, when j = 2 the subarray A[1...j-1] is sorted because A[1] contains a single element.

Maintenance

With each iteration of the for loop, the inner while loop shifts elements to the right until the current key can be inserted into the array at the correct location.

Termination

The for loop terminates when j > A.length in other words, the end of the array has been reached. We can observe that if j = A.length then A[1...j] will be sorted because the loop invariant was maintained. We can then conclude that the algorithm is correct.

Controlled not gate

Controlled not gates are only applicable in two qubit systems, so it is worth covering how that changes things.

In Quantum logic gates we dealt only with one qubit systems that has the basis states of \(| 0 \rangle\) and \(| 1 \rangle\) as described in Qubit state.

Now we have two bit systems which essentially add a bit to each possible basis state. So with a doubling of qubits, we have quadrupled the power of our system!

The states are \(|00\rangle\) \(|01\rangle\) \(|10\rangle\) and \(|11\rangle\). Each of which can have superpositions.

\( \alpha|00\rangle + \beta|01\rangle + \gamma|10\rangle + \delta|11\rangle \)

The normalaztion condition still exists so the squares of each scalar must add up to \(1\).

The controlled not gate, or CNOT, acts only on one qubit. There is the control qubit (the qubit that must satisfy a condition for the NOT action to occur) and the target qubit (the qubit that is modified).

A CNOT manipulation would perform the following:

\begin{align} |00\rangle \rightarrow |00\rangle\\ |01\rangle \rightarrow |01\rangle\\ |10\rangle \rightarrow |11\rangle\\ |11\rangle \rightarrow |10\rangle\\ \end{align}

Using multiple gates, such as a Hadamard gate followed by a CNOT gate leads to entanglement (not sure exactly what that means) and allows for interesting, unique processing.

Additionally, the control qubit can be modified. (What does this allow?)
